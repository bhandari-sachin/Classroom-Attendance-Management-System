# Sprint 3 Planning Report

## Sprint Number & Dates
- **Sprint:** Sprint 3
- **Start Date:** 10 February
- **End Date:** 03 March

---

## Sprint Goal
The goal of Sprint 3 was to extend the functional prototype of the QR-based attendance system,
integrate a CI/CD pipeline using Jenkins, strengthen automated testing and code coverage with JaCoCo,
and prepare a working local Docker image.

This sprint focused on transforming the system from a structural prototype into a feature-complete, testable, and continuously integrated application ready for functional review.
---

## During Sprint 3, the team aimed to:
- Extend and solidify backend and frontend functionality

- Implement user authentication and improve system workflows

- Integrate CI/CD automation

- Improve automated test coverage

- Prepare the system for end-to-end demonstration

- Build and test a local Docker image

---

- Fixe logic bugs identified during testing

- Established a complete CI/CD pipeline using Jenkins.
## Pipeline Stages

# Code Checkout

- Retrieves latest code from Git repository (main branch)

# Build Stage

- Compiles the project using Apache Maven

# Unit Testing

- Executes unit tests using JUnit

# Code Coverage

- Generates coverage reports using JaCoCo

## Definition of Done
A task is considered **Done** when:
- It satisfies the requirements defined in the sprint plan
- Code is committed and pushed to the project repository
- The database schema is implemented and documented
- Initial UI screens are functional
- Unit tests are written and passing
- Relevant documentation is updated
- Code coverage is generated and reviewed
