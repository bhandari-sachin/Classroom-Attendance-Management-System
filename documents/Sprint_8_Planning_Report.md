## Sprint Goal
The goal of Sprint 8 is to bring the project to a polished, demonstrable state in preparation for the final sprint review. This sprint focuses on delivering a clear and structured project presentation, documenting testing insights from heuristic evaluation and user acceptance testing (UAT), tracking sprint progress through review reports, ensuring code quality through code reviews and static analysis, and producing comprehensive system documentation including UML diagrams, ERD diagrams, and an updated README.

- Ensured Code Quality

## Planned Tasks / Breakdown
- Prepare and rehearse a structured 25-minute project presentation for the sprint review
- Document heuristic evaluation findings with identified usability issues and severity ratings
- Record and organise UAT results, capturing user feedback and mapping to requirements
- Write and publish the Sprint 8 review report on Trello/GitHub
- Conduct peer code review and document findings with suggested improvements
- Run static code analysis tools and include results in project documentation
- Create or update UML diagrams (class, sequence, use case) reflecting final system design
- Create or update ERD diagrams to accurately reflect the current database schema
- Update the project README with setup instructions, architecture overview, and usage guide
- Commit all documentation and code changes to GitHub and update Trello board
