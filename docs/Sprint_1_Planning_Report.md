# Sprint 1 Planning Report

## Sprint Number & Dates
- **Sprint:** Sprint 1  
- **Start Date:** 13 January  
- **End Date:** 26 January  
- **Sprint Duration:** 2 weeks  

---

## Sprint Goal
The goal of Sprint 1 is to establish the foundation for the software project in alignment with Week 1 and Week 2 course objectives. This sprint focuses on team formation, project selection, creation of the project vision and plan, introduction to software engineering and DevOps practices, and installation and setup of the required development tools.

---

## Selected Product Backlog Items
- Team formation and role allocation  
- Project topic selection  
- Project vision and high-level project plan  
- GitHub repository creation and configuration  
- Trello board setup for sprint and task management  
- Introduction to Software Engineering concepts  
- Overview of DevOps practices  
- Installation and setup of required development tools  

---

## Planned Tasks / Breakdown
- Form project team and assign responsibilities  
- Select and document project topic  
- Create project vision and initial project plan  
- Set up GitHub repository and add team members  
- Create and configure Trello board for sprint tracking  
- Attend lectures on Software Engineering and DevOps  
- Install and configure required tools as part of in-class assignment  
- Ensure development environment is ready for future sprints  

---

## Team Capacity & Assumptions
- Team members are available on a part-time basis due to academic workload  
- Sprint duration is fixed at two weeks in accordance with the course schedule  
- Team members are at an introductory level with DevOps practices  
- Collaboration and communication take place using GitHub, Trello, and online meetings  

---

## Definition of Done
A task is considered **Done** when:
- It satisfies the requirements defined in the sprint plan  
- Project documentation (vision, plan) is completed  
- Tools and environments are successfully installed and configured  
- Relevant changes are committed to GitHub  
- Tasks are updated and marked as “Done” on the Trello board  
- Sprint objectives aligned with Week 1 and Week 2 lectures are achieved
